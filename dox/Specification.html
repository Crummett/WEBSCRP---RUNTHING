Package.json

The project relies on 4 packages: ‘body-parser’, ‘express’, ‘mysql’, and ‘ejs’:

  • Body-parser - Used to interpret post requests.
  • Espress - Used to create the api. The heart of the website.
  • Mysql - Used to connect and send queries to and from a mysql database.
  • Ejs - Embedded JavaScript, used to insert data from the databases into web pages.

Server.js

As with the majority of Node.js websites, the website is structured around the server.js file, containing my api.

Each address has a get response, which gets the relevant info from the database via a query within the functions.js file, before rendering the relevant template.

There are also put responses, which take ajax posts from client-side JavaScript files and uses the relevant function from the functions.js file to post to the database.

Functions.js

Contains the functions used to make queries to the database.

Opens the connection to the database at the top. If the database does not exist it runs the createDatabase() function to run the necessary queries.

Module.exports at the bottom allows the functions to be used within the server.js file.

/views/

Contains the templates for the website pages.

These exist as HTML files with a .ejs extension. The <%= %> tags put variables from the data passed in when rendered to the browser.

/public/

Contains the stylesheet for the website, as well as the favicon

/public/lib

Contains the client-side JavaScript.

This chiefly concerns ajax posts for form data, and the relevant javascript for the Google Maps API.

Each file contains the functions for the identically-named .ejs file.

The maps uses Google’s geocoder to turn the address supplied for the event/club into the correct JSON, which the map then uses to make a marker and centre on it.

The Future

  • Currently it lacks any kind of login feature. The intention is to at some point use OAuth 2. Currently anytime the user performs an action, they have to select the user they would like to perform it. This would obviously not be suitable for a public website.
  • I’ve started trying chat between users, but was unable to integrate anything in time. Would be fairly simple to do, especially based on the framework used for posting to club walls.
  • An invite/RSVP system for events.
  • Further refinements to clubs and events, as detailed in the mark scheme. Stuff like pinning posts.
  • League tables and more competitive structure.
